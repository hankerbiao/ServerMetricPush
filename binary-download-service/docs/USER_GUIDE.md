# Binary Download Service 用户使用指南

## 概述

本文档介绍 Binary Download Service 的使用方法，包括 Web 界面的操作指南和常见使用场景。

## 访问界面

启动服务后，可通过以下地址访问：

| 地址 | 功能 |
|------|------|
| http://localhost:8080 | 文件管理控制台 |
| http://localhost:8080/agents | 节点状态监控 |

---

```
├─────────────────────────────────────────────────────────┤
│  文件列表                                               │

└─────────────────────────────────────────────────────────┘
```

### 上传文件

1. 点击上传区域或直接将文件拖拽到上传区域
2. 支持的文件格式：

   | 程序 | 文件名格式 | 示例 |
   |------|------------|------|
   | node_exporter | `{version}-{os}-{arch}.tar.gz` | node_exporter-1.5.0-linux-amd64.tar.gz |
   | node-push-exporter | `{version}-{os}-{arch}.tar.gz` | node-push-exporter-1.0.0-darwin-arm64.tar.gz |
   | 安装脚本 | install.sh / uninstall.sh | install.sh |

3. 上传成功后，文件会显示在文件列表中
4. 如上传同名文件，会自动替换旧版本

### 下载文件

在文件列表中，点击文件名旁边的下载按钮（↓），或使用复制功能获取 curl 命令。

### 删除文件

点击文件行中的删除按钮（🗑），确认后即可删除文件。

### 筛选文件

使用顶部的下拉菜单按程序类型筛选：

- **全部** - 显示所有文件
- **node_exporter** - 仅显示 node_exporter
- **node-push-exporter** - 仅显示 node-push-exporter
- **install-script** - 仅显示安装脚本

### 节点状态说明

| 状态 | 含义 | 指示灯 |
|------|------|--------|
| online | 节点正常运行 | 🟢 绿色 |
| degraded | 节点有异常（部分指标失败） | 🟡 黄色 |
| offline | 节点已离线 | 🔴 红色 |

### 节点注册

node-push-exporter 启动时会自动向服务注册。注册时需要提供：

```json
{
  "agent_id": "唯一标识符",
  "hostname": "主机名",
  "version": "程序版本",
  "os": "操作系统",
  "arch": "架构",
  "ip": "IP地址",
  "pushgateway_url": "Pushgateway 地址",
  "push_interval_seconds": "推送间隔（秒）",
  "node_exporter_port": "node_exporter 端口",
  "node_exporter_metrics_url": "node_exporter 指标 URL",
  "started_at": "启动时间"
}
```

### 心跳机制

- 节点每 30 秒发送一次心跳
- 超过 90 秒未收到心跳，状态变为 offline
- 心跳包含节点状态、错误信息、推送结果等

## 使用示例

### 示例 1：通过 Web 下载文件

1. 打开 http://localhost:8080
2. 在文件列表中找到目标文件
3. 点击下载按钮下载文件

### 示例 2：使用 curl 下载

在文件列表中点击复制按钮，获取下载命令：

```bash
curl -O http://localhost:8080/download/node_exporter-1.5.0-linux-amd64.tar.gz
```

### 示例 3：监控节点状态

1. 打开 http://localhost:8080/agents
2. 查看所有已注册节点的状态
3. 点击节点查看详细信息和事件历史

### 示例 4：API 上传文件

```bash
curl -X POST http://localhost:8080/api/upload \
  -F "file=@node_exporter-1.5.0-linux-amd64.tar.gz"
```

### 示例 5：API 列出文件

```bash
# 列出所有文件
curl http://localhost:8080/api/files

# 筛选特定程序
curl "http://localhost:8080/api/files?program=node_exporter"
```

## 常见问题

### Q: 文件上传后不显示

1. 检查文件名格式是否正确
2. 检查浏览器控制台是否有错误
3. 尝试刷新页面

### Q: 节点显示 offline

1. 检查节点服务是否在运行
2. 检查网络连通性
3. 检查节点配置的心跳间隔

### Q: 如何重启服务

```bash
# 停止当前服务 (Ctrl+C)
# 重新启动
./run.sh
```

### Q: 如何查看日志

服务日志直接在终端输出。生产环境可配置日志输出到文件。
